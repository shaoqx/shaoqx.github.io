Rosetta Notes

Notes of using Rosetta

Minimization

A simple version of minimization steps from the RosettaCommons tutorial

We use the Rosetta application, <path_to_Rosetta>/main/source/bin/minimize.default.linuxgccrelease, to run a Rosetta score based minimization

Set Flags

Rosetta use flags in a flag file (or in the command line) to control to behavior of the program.
Flags we use here are:

  • in:file
    This section is general for reading a structure into Rosetta.
    -s supplies the PDB file.
    -extra_res_fa supplies params files for structures that is not canonical residues.
  • The run section
    -min_type specifies the minimization algorithm (default: lbfgs_armijo_nonmonotone)
    -min_tolerance specifies the convergence tolerance (defination of the tolerance)

fake_rotlib.py-based solution

@Eric Bell has an improved protocol

Generate params file

  1. Get fake_rotlib.py and put it under {rosetta_home}/main/source/scripts/python/public/
    * Note that if you are using Rosetta <= 3.13, detele the entire public folder and replace with a latest on from GitHub main
  2. Install rdkit by conda install -c conda-forge rdkit
  3. Prepare the NCAA
    • build/find the molecule and save it as .sdf
      * make it valance-closed.
      * Gaussian .sdf dont work with RDKit. Dont use it.
    • specifiy correct formal charge in the file
      For sdf files the formal charge is specified by 2nd column after the element symbol.
      SDF use a confusing format for the value of the charge -> defination
  4. Run the script
    python {rosetta_home}/main/source/scripts/python/public/fake_rotlib.py -i /{path}/xxx.sdf
    It will generate a .params file and a _rotamer.pdb which can be used in Rosetta
    (2023.10.4) Using relative path will cause a bug.

Preparing PDB for Rosetta

QZ Shao
Hi! How should I prepare a PDB with metal and modified AA for Rosetta relax? I tried to use clean_pdb.py but it always remove my metal and ncaa. I saw document recommand clean_pdb_with_ligand.py but I cant find it under tools/ . Thanks in advance!

rmoretti
“Cleaning” is generally used to remove everything that isn’t canonical amino acids. There’s been a fair amount of effort put into making Rosetta more robust to PDB read in. These days there shouldn’t be a need to >”clean” PDBs prior to using them with Rosetta. – At most you may just need to edit them to remove residues which you don’t want to model (e.g. crystallization adducts). But that’s mostly a manual and selective >approach, rather than the wholesale removal that the clean scripts take.

QZ Shao
Got it! Also one note to this is I found I need to change HID HIP etc. back to HIS if the PDB is used for/in Amber.

rmoretti
Yes. Rosetta automatically switches between the two neutral tautomers of histidine. (Note that HIP is positively charged for Amber, while HIS is strictly neutral for Rosetta.)
