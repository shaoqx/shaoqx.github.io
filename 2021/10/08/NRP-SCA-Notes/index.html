NRPS-SCA Notes

Record workflow of the project of NRP-SCA in the 2nd rotation in the Walker group.

Background

NRP are important natural products that serve as important drugs.
NRP synthase usually have a long gene that encode a large enzyme.
The large enzyme can be divided into repetitive functional regions with a unit of 3 : C(condensation) - PCP(Peptidyl Carrier Protein) - A(activation) that corresponding to each amino acid. And in the end a TE indicating the termination.
Among 3 or more, the A functional domain determine which amino acid is installed.


Relate Video

To do mutations on NRP, the corresponding C domain of the NRPase is engineered. But most attempt failed finding that C domain is always coupled with A and will lose reactivity.
We need to figure out which positions are coupled and find a place to edit. Since grafting A domain will need replacing all the correlated parts and avoiding parts that relate with another A.
A fuse point was reported previously that the C domain partially relate to the previous A and partially relate to the latter A. But replacing sequences between fuse points, mutagenesis is done.

Potential

The coupling in NRPase is domain coupling inside of a large enzyme
This kind of coupling can also be extend to the idea of enzyme-enzyme coupling in a gene cluster for a specific natural product. For enzyme engineering of a larger scale.

Workflow

1. Scrap annoted gene data from DB

Domain Knowledge

Architectures of gene DataBase

Gene is stored in the hierarchy of [genome - gene_cluster - gene]

  • genome is a complete set of DNAs of a specific species.
  • gene_cluster is a section of genome (with vague edges) that genes in it share a common goal (e.g. a natural product synthesis)
  • gene is a DNA fragment that determined by initial codon and terminal codon that corresponding to a specific protein

antiSMASH gene cluster page

The color full bar with arrows shows the whole gene clusters
The green bar shows the part that related to NRPS and the dark green shows the core part of the NRPS (containing A&C)
Annotation of each NRPS related gene are in the table and complete unit that accont for a specific AA is cover with gray.

gbk file format

  • every gene is renumbered within gcluster in the data line
  • There are 3 levels of data:
    • First block data like ACCESSION start at 1st position: properties of gene cluster
    • Second block data like gene start at 5th position
    • Third block data like /gene_functions may not complete a data line before the line_feed
  • The order of the aSDomian is reversed to the order of protein

Steps

antiSMASH-fetch.py (github link place holder)

  • Download the csv file from query.
  • Decode the csv file to get (GCX_id, start_id, end_id) (The links themselves are not working)
  • Match s_id & e_id in the GCX_id genome page

2. MSA

HMMer

MSA is done by HMMer.
HMMer uses the HMM algorithm that do MSA and homology probablity base on a background distribution MSA.

It is faster and more accurate when there’s a existing homology MSA (library).

Steps

  • Background MSA: download one for A and C domain each from Pfam as .hmm file
    This is the a “seed” alignment of a small number of representative sequences, and this profile HMM can be used to efficiently align any number of additional sequences.
    This is in fact how the PFAM database is updated as the main SPTREMBL database increases in size.The PFAM seed alignments are (relatively) stable from release to release

  • Run hmmalign:
    hmmalign --outformat A2M -o NRPS_AnpC_A.a2m AMP-binding.hmm NRPS_AnpC_A.fasta
    It will export the resulting MSA of the .fasta file using the .a2m format as required by the following pySCA.

3. SCA

Statistical Coupling Analysis is done by pySCA.

Steps

1. Precondition

MSA: need a MSA of >100 sequence.
(done by HMMer)

2. Pre-processing (scaProcessMSA)

pySCA (scaProcessMSA) will remove some bad sequences.
Note: One can install fasta36 by adding its path to PATH instead of moving it into /usr/local/

  • File conversion: scaProcessMSA need .fasta input.

    • Use .a2m as the output of hmmalign
      The output a2m still seems inconsistant with the .sto result and the sequence length is inconsistant
      Need some post-processing after hmmalign to keep all sequence in the same length!! (How?)

    • Convert with the Biopython package

      This seems not giving an MSA since there’s no gap in the output

    • Convert with esl-reformat -u --informat stockholm -o *fasta fasta *sto
      This also do not contain gaps nor phrased the ‘.’ from .sto

  • Concatnate MSA

Use MSA_tool.py to cat A and C MSA base on their labeled id.

  • Process MSA Use default refsequence indicated in the stdout and numbering from 1 of the MSA
    (The desired fuse point should be a MSA position)
    This will result a python database {name}.db and a processed MSA

In the command:

  • -a: Input_MSA.fasta (the alignment to be processed, typically the headers contain taxonomic information for the sequences).

  • Reference Sequence
    The reference sequence can be specified in one of four ways:

      By supplying a PDB file - in this case, the reference sequence is taken from the PDB (see the pdb kwarg, *-s*, *-c*, *-b*)

  By supplying a reference sequence directly (as a fasta file - see the refseq kwarg)

  By supplying the index of the reference sequence in the alignment (see the refseq kwarg)

  If no reference sequence is supplied by the user, one is automatically selected using the scaTools function chooseRef.

  • Position numbering
    The position numbers (for mapping the alignment, because truncation and deletion will be done by scaProcessMSA) can be specified in one of three ways:

      By supplying a PDB file - in this case the alignment positions are mapped to structure positions

  By supplying a list of reference positions (see the refpos kwarg)

  If no reference positions are supplied by the user, sequential numbering (starting at 1) is assumed.

3. SCA (scaCore)

This will write
Sequence correlations:
the global sequence similarity matrix for the alignment
the eigenvectors (and independent components)
Positional correlations:
Compute the single-site position weights and positional conservation values
Compute the dimension-reduced SCA correlation matrix
Compute Ntrials of the randomized SCA matrix, and the eigenvectors and eigenvalues associated with each

Visualize SCA result

Use a visualization script and the SCA result in the python pickle db
The pickle db store the result as a dictionary. (pickle.load())
The dimension-reduced SCA correlation matrix is stored in db[‘sca’][‘Csca’]

  • The SCA matrix with lines indicating A/C sections.
