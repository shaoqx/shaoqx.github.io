Progdyn2017的部署与使用

本文介绍了Quasi-classical动力学软件Progdyn2017的安装，运行，监控以及数据分析(Chinese Version)

安装

  1. 将Progdyn文件夹放置在~/bin/progdyn/progdyn_2017路径下
    • 如果放在其他位置，需要编辑 makedynjob 与 setup_dyn以适配自定义路径
    • 系统要求：Linux/Unix
  2. 修改文件夹权限chmod -R 755 ~/bin/progdyn/progdyn_2017

在其他服务器上的安装

  1. 修改progdynstarterHP中的55 56行 设置为本服务器加载Gaussian的方式
  2. 修改makedynjob 130行之后的提交脚本部分 设置为本服务器的格式

运行

这里以Hoffman2的集成任务管理环境(SGE)为例

  1. 建立项目文件夹mkdir $Project

  2. 计算对象过渡态的HPmode频率(使用freq=hpmodes关键字)，将结果放置于项目文件夹中 注意该计算不能使用nosymm

  3. 运行设置脚本 sh ~/bin/progdyn/progdyn_2017/setup_dyn [gau_freq_calc.out] [directory]

    • **[gau_freq_calc.out]**是过渡态频率文件
    • **[directory]**是项目路径，如果是当前路径则可以简化为.
    • 该过程会将频率文件改名并复制两个配置文件(proganal progdyn.conf)到项目路径

  4. 编辑配置文件

    progdyn.conf 常用设置

    • 设置需要的计算资源(核心数/内存) 该配置将体现在提交脚本和gaussian输入文件中
    • 设置使用的理论层级-method 溶剂化模型-method3 必要关键字-method4 体现在Gaussian输入文件中
    • 设置numimag 对中间体采样的时候设置为0
    • 设置title 体现为提交脚本中的任务标题 设置以在队列中方便查看

    需要注意progdyn2017只支持到D2，不支持D3矫正

    proganal 常用设置
    设置终止条件（END{}部分）

    • 定义监控的键长：设置名称和值 更改printf中的对应关系（数量/名称）
    • 设置单侧最大轨线时长
    • 设置终止键长限制 修改对应的终止信息

  5. 提交任务

    运行sh ~/bin/progdyn/progdyn_2017/makedynjob

    • 设置同时运行的轨线数n
    • 决定是否现在提交 (如果此处选no之后可以在项目路径下手动提交qsub ./n#/subdynjob.n#.job)

监控

可以通过每个轨线文件夹中的dynfollowfile观察监控键长的变化
可以通过traj文件观察轨线（下载并改成.xyz）

维护

Progdyn的每个轨线文件夹在跑完一条轨线后会自动开始下一条
对于有限的任务时间，需要在任务结束后重新提交所有.job文件，Progdyn会自动继续所有轨线运算 由于任务开始和结束的时间可能不同，需要注意一定不要重复提交/运算同一个轨线任务 我的Github上有我写过的一个自动提交跑完的部分轨线的脚本（只适用于Hoffman2）

结构的简谐分布区域分析

在实际的体系中不同的中间体、过渡态不以稳定点的固定结构存在，而是体现为在稳定点附近的基于振动熵最大化的分布
Progdyn中可以通过进行normal mode sampling得到这个分布的采样 从而可以被表征为一个区域
通过对该区域的数学定义可以分析轨线在该区域的行为（是否经过，停留时间, etc）

  • 采样区域的数学定义
    normal mode sampling得到的一组结构往往被体现为在几个关键键长上的分布 （例如：ambimodal中的2nd竞争键）
    在这种情况下体现为在一个2d面上的采样，并且由于这两个键的运动往往耦合所以分布体现为有一定相关性的斜椭圆（如果完全独立则为高斯正圆）
    数学定义
    通过一个区域拟合该结构可能的分布 – 一定置信度下包含全部可能的采样点 – 置信椭圆
    根据采样点的置信椭圆的求解可以通过Pearson相关系数法得出
    通过解得的椭圆方程(不等式)可以判断一个点是否在椭圆内
  • 进行区域分析
    对单条轨线可以得到停留时间
    对多条轨线可以得到平均停留时间，经过区域的轨线比例
    相关脚本：https://github.com/shaoqx/CalcKit/blob/master/Progdyn2017_tool/Analysis/Region_analysis.py
  • 采样区域作图
    可以通过matplotlib对椭圆进行作图之后用ps叠加在轨线上
    相关脚本：https://github.com/shaoqx/CalcKit/blob/master/Progdyn2017_tool/Plot/sample/Progdyn_plot_sep.py

使用xtb方法

通过Gaussian的external接口调用外部程序xtb完成力的计算

基于gau_xtb程序
http://sobereva.com/soft/gau_xtb/

1. 安装xtb

https://xtb-docs.readthedocs.io/en/latest/contents.html

2. 下载使用gau_xtb程序

  • 修改 xtb.sh 设置xtb的运行参数
    核数: OMP_NUM_THREADS 和 MKL_NUM_THREADS
    溶剂: --gbas后的溶剂名
  • 将 genxyz extderi xtb.sh 移动至项目文件夹
  • 调整权限 chmod 755 genxyz extderi xtb.sh

3. 部署Progdyn

freqinHP变化：

  • 过渡态和HPmodes也必须是在xtb层级下计算的否则会一直采样失败
    在hof2上可以通过subg16c418x完成计算:提供gjf和xtb.sh
    progdyn.conf额外修改:
  • 核数设置为 1 防止高斯并行
  • 计算方法 设置为external=’./xtb.sh’
  • 溶剂 删除高斯的溶剂(直接注释掉method)

4. 部署后修改

subdynjob.nX.job：

  • 修改核数至与xtb中一致，如：
    for i in n*;do sed -i 's/shared* 1/shared* 4/' $i/subdynjob.$i.job;done

  • 移动gau_xtb组件至高斯运行路径
